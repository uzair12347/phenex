Welcome to the Phenex VIP project. I'm Ben, product owner. Happy to have you on board.

*What we're building*

A Telegram-based VIP membership system for traders. Three parts:

1️⃣ *Backend API* (Node.js + PostgreSQL) – all the logic: customer sync, rule engine, CRM, integrations
2️⃣ *Admin Dashboard* (React) – full control panel for our team
3️⃣ *Telegram Mini App* – the onboarding gate every user goes through before joining the VIP group

The data comes from TauroMarkets (our partner broker). They expose a REST API that tells us who's actively trading, what their balances are, and whether they're depositing or withdrawing. Our job is to turn that data into automated actions – reminders, ban/unban, segment assignments, CRM tasks – via a configurable rule engine.

⸻

*Your first step*

I've attached the full onboarding document. Read it top to bottom before touching any code. Everything is in there: architecture, DB schema, all API endpoints, how the rule engine works, how the Tauro API integration works, and the known quirks.

The codebase is already written. Your job in this first phase is to understand it, set it up locally, and complete the test task described at the end of the onboarding doc (Section 12).

⸻

*Test task (Section 12 of the doc)*

Add a "Reactivation Opportunity" rule via the API. No UI work needed. Just:
- Define the rule (conditions + actions)
- Run a dry-run
- Write a short explanation of two things (in the doc)

The test task does NOT need live Tauro credentials. Everything runs locally with dummy data.

⸻

*Technical setup*

You'll need: Node.js 20, PostgreSQL 16, Redis 7 (or Docker).

```
cd backend
cp .env.example .env
npm install
npm run migrate
npm run seed
npm run dev
```

Default admin login after seed:
Email: admin@phenex.com
Pass: admin123

For the admin frontend:
```
cd frontend-admin
npm install
npm start
```

The Mini App is a single HTML file – no build step. Just open frontend-miniapp/index.html or serve it with `npx serve`.
